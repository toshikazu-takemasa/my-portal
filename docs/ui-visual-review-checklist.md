# UI Visual Review Checklist (Persona-inspired)

最終見た目確認用のチェックリストです。  
対象: `main` の最新デザイン（赤/黒/白・非対称・鋭角）

## 1) Desktop（1440px前後）

### A. 全体トーン
- [ ] 背景が暗基調で、赤アクセントが主役になっている
- [ ] 白カードが黒背景に対して十分なコントラストを持つ
- [ ] 旧パステル調（ベージュ/淡オレンジ）の違和感が残っていない

### B. レイアウト/構図
- [ ] 3カラム構成で中央カラムの存在感が最も高い
- [ ] カードの段差（シャドウ）で前後関係が視認できる
- [ ] 非対称な配置でも情報導線が迷子にならない

### C. 造形（Persona感）
- [ ] カードの鋭角カット（clip-path）が意図通り表示される
- [ ] 斜線・切り抜き感・ハーフトーンが過剰にうるさくない
- [ ] タブ、ボタン、吹き出しが角丸ではなくシャープ寄り

### D. タイポグラフィ
- [ ] `My Portal` 見出しが十分に目立つ
- [ ] サブテキスト（日時等）が読めるが主張しすぎない
- [ ] ラベル（USER ANALYTICS 等）の字間・太さが統一されている

### E. 操作部品
- [ ] Hover時の挙動（色反転/移動）が統一されている
- [ ] ボタン/入力欄の境界線が視認しやすい
- [ ] 設定モーダルの視認性と階層が明確

## 2) Mobile（390px前後）

### A. 情報密度
- [ ] 初期表示でヘッダー→分析帯→主要カードの流れが自然
- [ ] カード内の余白が狭すぎず、スクロール負荷が高すぎない
- [ ] 文字サイズが読める（特に日付・補助ラベル）

### B. Bottom Nav
- [ ] アイコンとラベルが被らず判読可能
- [ ] アクティブ状態（赤）が直感的に判別できる
- [ ] Safe Area（端末下部）で操作不能領域がない

### C. 右サイドメニュー
- [ ] 開閉アニメーションがスムーズ
- [ ] オーバーレイ表示中の背面誤タップが発生しない
- [ ] 画面幅が狭い端末でも内容が切れない

## 3) 機能表示との整合

- [ ] 日報タブ/編集タブで視認性が崩れない
- [ ] AIチャット吹き出し（user/ai）の区別が明確
- [ ] Google Calendarプレースホルダのボタンが読みやすい
- [ ] チェックリスト/タスクカードの行間と境界が適切

## 4) 最終判定（Go / Fix）

- [ ] ブランド感（P5インスパイア）を感じる
- [ ] 実運用で読みにくい箇所がない
- [ ] PC/モバイルともに致命的な崩れなし
