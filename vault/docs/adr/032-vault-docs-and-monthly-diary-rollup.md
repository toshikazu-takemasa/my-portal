# ADR 032: docs を vault 配下へ移動・日記の月次まとめ運用

**日付**: 2026-07-10
**状態**: 採用
**関連**: ADR 031（ディレクトリ構造を work-vault と統一）

## コンテキスト

- ADR 031 で「portal-app + vault」の2階層に統一したが、docs/（ADR・画像・旧設定）だけが
  リポジトリ直下に残っていた
- vault/diary/ に日別ファイルが120以上たまり、一覧性が悪く GitHub API の
  ディレクトリ取得も重くなっていた（1月分のみ月次まとめ済みで運用が不統一だった）

## 決定

1. **docs/ → vault/docs/ へ移動**
   - 設計記録・画像もデータとして vault に集約する
   - アプリからの参照は `vault/docs/adr`（API）・`../vault/docs/images/avatar.png`（相対）に更新
2. **日記は「当月=日別、過去月=月次まとめ」の運用に統一**
   - 過去月は `YYYY-MM.md`（`# YYYY年M月` + `## YYYY年M月D日` 見出し・`---` 区切り）に統合
   - 2026-01〜2026-06 の日別116ファイルを統合・削除（全行が月次に包含されることを機械検証済み）
   - 当月（2026-07）は日別のまま。月が明けたら同様にまとめる
3. **ファイル名の整理**
   - `diary/2024-04-summary.md` → `diary/2024-04.md`（月次パターンに統一）
   - `diary/未整理.md` → `diary/テンプレート.md`（実体は日次テンプレート）
   - `knowledge/批評訓練` → `knowledge/批評訓練.md`（拡張子付与）

## 影響

- アプリの日次読み書き（`vault/diary/YYYY-MM-DD.md`）は当月ファイルのみ対象のため無影響
- AI チャットの日記参照は過去月について月次ファイルを読む形になる
- `vault/docs/data/portal-config.json` は旧版設定（pillars 等）を含むため削除せず温存
